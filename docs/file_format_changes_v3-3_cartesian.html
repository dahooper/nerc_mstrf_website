--- layout: default title: Proposed changes to v3 Cartesian files --- In preparation for the re-processing of the entire MST radar observation archive, a few small changes have been made to upgrade the current version 3.2 signal processing scheme to a version 3.3 scheme:
  • A correction has been made to the (time-continuity) reliability flagging scheme, which used to cause problems when the radar cycle format was changed (i.e. infrequently).

  • The assumed value for the speed of light, which is used to calculate the range and altitude of radar range gates, has been changed from 3.0e8 to 299,792,458.0 m/s. This will be relatively insignificant for the ST-mode observations, leading to a maximum difference in the calculated ranges and altitudes of 15 m. However, it leads to differences of up to 65 m, i.e. almost half a range gate, in the M-mode.

  • A change will be made to the bit-encoding of the reliability details variables in order to remove a redundant piece of information. However, there will still be an additional binary flag which simply indicates whether or not a parameter value is judged to be reliable.

  • The Variable and Global Variable names in the netCDF files will be changed slightly in order to reduce ambiguities and to better reflect the relationships between parameters. This logic is most apparent when the new names are viewed as a group. Tables are also provided which interlace the current and new names. The new names may be changed slightly compared to those proposed here. Final details will be included in a separate page once the reprocessing begins.
Tables are available for the following:

  • Version 3.3 Variable names
  • Interlaced version 3.2 and version 3.3 Variable names
